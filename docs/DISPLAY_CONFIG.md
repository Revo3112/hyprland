## Config Structure Overview

### Directory Architecture

```
~/.config/hypr/
├── hyprland.conf              # Main entry point (sources other files)
│
├── hyprland/                  # ⚠️ UPSTREAM CORE FILES - DO NOT MODIFY
│   ├── env.conf              # Environment variables (Wayland, Qt, Theme)
│   ├── execs.conf            # Core startup (quickshell, clipboard, keyring)
│   ├── general.conf          # Animations, blur, gestures, input
│   ├── keybinds.conf         # All default keybinds
│   ├── rules.conf            # Window/layer rules
│   ├── colors.conf           # Theme colors (auto-generated by matugen)
│   └── scripts/              # Upstream helper scripts
│
├── custom/                    # ✅ SAFE TO MODIFY - Your customizations
│   ├── env.conf              # Custom environment variables
│   ├── execs.conf            # Custom startup apps
│   ├── general.conf          # Custom monitor/layout settings
│   ├── keybinds.conf         # Custom keybinds
│   └── rules.conf            # Custom window rules
│
├── monitors.conf             # nwg-displays managed monitor config
├── workspaces.conf           # nwg-displays managed workspace config
└── scripts/                   # Custom user scripts
```

### Config Loading Order

Files are loaded in this order (later files override earlier):

1. **Upstream defaults:** `hyprland/*.conf`
2. **Custom overrides:** `custom/*.conf`
3. **Display tools:** `monitors.conf`, `workspaces.conf`

### Files Safe to Modify

| File | Purpose | Status |
|------|---------|--------|
| `custom/env.conf` | Custom environment variables | ✅ Safe |
| `custom/execs.conf` | Custom startup apps | ✅ Safe |
| `custom/general.conf` | Monitor & general settings | ✅ Safe |
| `custom/keybinds.conf` | Custom keybinds | ✅ Safe |
| `custom/rules.conf` | Custom window rules | ✅ Safe |
| `monitors.conf` | nwg-displays output | ✅ Safe |
| `workspaces.conf` | nwg-displays output | ✅ Safe |

### Files NOT Safe to Modify (Upstream)

| File | Risk |
|------|------|
| `hyprland/*.conf` | Will be overwritten on upstream updates |
| `hyprland/scripts/*` | Core functionality may break |
| `hyprland/colors.conf` | Auto-generated by matugen theme |

## HDMI Monitor Configuration

### end-4/dots-hyprland Approach

end-4/dots-hyprland **tidak menyertakan** display management tool. Monitor dikonfigurasi secara manual di `custom/general.conf`.

---

### Method 1: HyprDynamicMonitors ⭐ RECOMMENDED for Laptops

**GitHub:** https://github.com/fiffeek/hyprdynamicmonitors

**Keunggulan:**
- ✅ Built specifically for Hyprland (native syntax)
- ✅ Auto-switch profile saat HDMI plug/unplug
- ✅ Power-aware (berbeda profile saat battery vs AC)
- ✅ Lid state detection (laptop lid open/close)
- ✅ TUI interface untuk profile management

**Installation:**
```bash
yay -S hyprdynamicmonitors-git
```

**Configuration:**
```bash
# Run TUI to create profiles
hyprdynamicmonitors

# Or run as daemon for auto-switch
hyprdynamicmonitors --daemon
```

**Add to `custom/execs.conf`:**
```ini
exec-once = hyprdynamicmonitors --daemon
```

---

### Method 2: nwg-displays (GUI - Like Windows)

**GitHub:** https://github.com/nwg-piotr/nwg-displays

**Installation:**
```bash
sudo pacman -S nwg-displays
```

**Usage:**
```bash
nwg-displays
```

**Features:**
- Drag & drop monitor positioning
- Mirror/extend mode toggle
- Resolution & refresh rate selection
- Saves to `monitors.conf` and `workspaces.conf`

**Note:** File `monitors.conf` dan `workspaces.conf` sudah di-source di `hyprland.conf` (end-4 setup)

---

### Method 3: shikane (Daemon - Alternative to kanshi)

**GitLab:** https://gitlab.com/w0lff/shikane

**Installation:**
```bash
sudo pacman -S shikane
```

**Configuration:** `~/.config/shikane/config`

```
# Profile: Laptop only
[[profile]]
name = "laptop-only"
    [[profile.output]]
    name = "eDP-1"
    enable = true
    mode = "1920x1200@60Hz"
    position = "0,0"

# Profile: HDMI Mirror
[[profile]]
name = "hdmi-mirror"
    [[profile.output]]
    name = "eDP-1"
    enable = true
    mode = "1920x1200@60Hz"
    position = "0,0"
    [[profile.output]]
    name = "HDMI-A-1"
    enable = true
    mirror = "eDP-1"

# Profile: HDMI Extend
[[profile]]
name = "hdmi-extend"
    [[profile.output]]
    name = "eDP-1"
    enable = true
    mode = "1920x1200@60Hz"
    position = "0,0"
    [[profile.output]]
    name = "HDMI-A-1"
    enable = true
    mode = "preferred"
    position = "1920,0"
```

**Add to `custom/execs.conf`:**
```ini
exec-once = shikane
```

### Method 4: Manual Config in Hyprland

Add to `custom/general.conf`:

#### Mirror Mode (Duplicate Screen)
```ini
monitor=eDP-1,1920x1200@60,0x0,1
monitor=HDMI-A-1,preferred,auto,1,mirror,eDP-1
```

#### Extend Mode (HDMI on Right)
```ini
monitor=eDP-1,1920x1200@60,0x0,1
monitor=HDMI-A-1,preferred,1920x0,1
```

#### Extend Mode (HDMI on Left)
```ini
monitor=HDMI-A-1,preferred,0x0,1
monitor=eDP-1,1920x1200@60,1920x0,1
```

#### HDMI Only (Laptop Off)
```ini
monitor=eDP-1,disable
monitor=HDMI-A-1,preferred,0x0,1
```

---

### Method 5: Quick Keybinds

Add to `custom/keybinds.conf`:

```ini
# Monitor profile keybinds
#! Display
##! Monitor Profiles
bind = Super+Alt, F1, exec, hyprctl keyword monitor "HDMI-A-1,preferred,auto,1,mirror,eDP-1" # Mirror
bind = Super+Alt, F2, exec, hyprctl keyword monitor "HDMI-A-1,preferred,1920x0,1" # Extend Right
bind = Super+Alt, F3, exec, hyprctl keyword monitor "HDMI-A-1,preferred,0x0,1" && hyprctl keyword monitor "eDP-1,1920x1200@60,1920x0,1" # Extend Left
bind = Super+Alt, F4, exec, hyprctl keyword monitor "eDP-1,disable" && hyprctl keyword monitor "HDMI-A-1,preferred,0x0,1" # HDMI Only
bind = Super+Alt, F5, exec, hyprctl keyword monitor "HDMI-A-1,disable" # HDMI Off
bind = Super+Alt, F6, exec, hyprctl keyword monitor "eDP-1,1920x1200@60,0x0,1" # Laptop Only
```

## Troubleshooting

### HDMI Not Detected

1. Check physical connection
2. Verify port status:
   ```bash
   cat /sys/class/drm/card1-HDMI-A-1/status
   ```
3. Reload Hyprland:
   ```bash
   hyprctl reload
   ```

### Phantom Monitor (HDMI still shows after disconnect)

Known Hyprland issue. Solutions:
```bash
# Disable the phantom monitor
hyprctl keyword monitor HDMI-A-1,disable
```

### Portal Issues

1. Restart portals:
   ```bash
   systemctl --user restart xdg-desktop-portal-hyprland xdg-desktop-portal
   ```
2. Re-login to Hyprland
3. Check portal status:
   ```bash
   systemctl --user status xdg-desktop-portal-hyprland
   ```

### Browser Screen Sharing

Use Wayland native flags:
```bash
--enable-features=UseOzonePlatform --ozone-platform=wayland
```

### Check Available Monitor Modes

```bash
hyprctl monitors all
```

## Technical Implementation Details

### HDMI Detection Method

| Approach | CPU Usage | Latency | Implementation |
|----------|-----------|---------|----------------|
| **Hyprland Events** (Current) | 0% | 0ms | Event-driven via `onRawEvent` |
| Timer Polling (Previous) | ~0.1% | 0-2000ms | Periodic check every 2s |

### How It Works Now

```
┌─────────────────────────────────────────────────────────────┐
│                    EVENT-DRIVEN FLOW                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. HDMI cable plugged in                                   │
│       ↓                                                     │
│  2. Kernel detects DRM change                               │
│       ↓                                                     │
│  3. Hyprland emits "monitoradded" event                     │
│       ↓                                                     │
│  4. Quickshell receives via Hyprland.onRawEvent()           │
│       ↓                                                     │
│  5. HdmiMonitorToggle updates instantly                     │
│                                                             │
│  RESULT: Zero CPU usage when idle, instant response         │
└─────────────────────────────────────────────────────────────┘
```

### Code Comparison

**Before (Polling - WASTEFUL):**
```qml
Timer {
    running: true
    interval: 2000      // Check every 2 seconds
    repeat: true        // Runs forever
    onTriggered: checkHdmiStatus.running = true
}
```

**After (Event-Driven - EFFICIENT):**
```qml
Connections {
    target: Hyprland
    function onRawEvent(event) {
        if (event.name === "monitoradded" || event.name === "monitorremoved") {
            checkHdmiStatus.running = true  // Only runs when needed
        }
    }
}
```

### Benefits

- ✅ **Zero CPU overhead** - No background polling
- ✅ **Instant response** - No 0-2 second delay
- ✅ **Battery friendly** - No wasted cycles
- ✅ **Professional approach** - Uses system events properly
